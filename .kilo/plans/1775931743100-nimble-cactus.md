# Plan: Fix Vercel Build - Missing Environment Variables in Turbo

## Problem

The Vercel build failed because Turbo doesn't pass environment variables to the build unless they're explicitly declared in `turbo.json`. The following vars are set on Vercel but missing from turbo.json:

- `SUPABASE_SERVICE_ROLE_KEY`
- `GOOGLE_CLIENT_ID`
- `GOOGLE_CLIENT_SECRET`

## Root Cause

The `build` task in `turbo.json` has `inputs` that doesn't include these environment variables. By default, Turbo doesn't pass env vars to the build - they must be explicitly listed.

## Solution

Add the required environment variables to the `build` task's `env` field in `turbo.json`:

```json
"build": {
  "dependsOn": ["^build"],
  "inputs": ["$TURBO_DEFAULT$", "package.json", "packages/*/package.json", ".env*"],
  "env": ["SUPABASE_SERVICE_ROLE_KEY", "GOOGLE_CLIENT_ID", "GOOGLE_CLIENT_SECRET"],
  "outputs": [".next/**", "!.next/cache/**", "dist/**"]
}
```

## Execution

1. Edit `turbo.json` to add the `env` array with the three env vars
2. Commit the change
3. Push to trigger a new build on Vercel

## Alternative (not recommended)

Add env vars to `globalEnv` in turbo.json if they should be available to all tasks.

## Phase 2: Add Realtime for Dashboard

### 2.1 Database setup

Add tables to `supabase_realtime` publication:

```sql
-- Enable realtime on landing tables
alter publication supabase_realtime add table landing.waitlist;
alter publication supabase_realtime add table landing.campaigns;
```

### 2.2 Create realtime hooks

Create `hooks/useRealtimeWaitlist.ts`:

```typescript
- Subscribe to `landing.waitlist` changes
- Emit events on INSERT/UPDATE/DELETE
- Cleanup channel on unmount
```

Create `hooks/useRealtimeCampaigns.ts`:

```typescript
- Subscribe to `landing.campaigns` changes
- Emit events on INSERT/UPDATE/DELETE
```

### 2.3 Update dashboard components

Dashboard currently is full Server Component. Strategy:

1. **Keep Server Component** for initial data fetch (SEO, fast first paint)
2. **Add Client Wrapper** that subscribes to realtime and triggers refresh

Options:

- A: Create `DashboardClient` that takes initial data + subscribes to updates
- B: Use `useRouter().refresh()` on realtime events to revalidate server data

**Recommended: Option B** — simpler, keeps Server Component benefits, revalidates on client.

```typescript
// components/dashboard/dashboard-realtime.tsx
'use client'
useEffect(() => {
  const channel = supabase
    .channel('dashboard')
    .on('postgres_changes', { event: '*', schema: 'landing', table: 'waitlist' }, () => {
      router.refresh()
    })
    .subscribe()
  return () => {
    supabase.removeChannel(channel)
  }
}, [])
```

### 2.4 Apply to all dynamic pages

| Page      | Tables to watch     |
| --------- | ------------------- |
| Dashboard | `landing.waitlist`  |
| CRM       | `landing.waitlist`  |
| Campaigns | `landing.campaigns` |
| Events    | `public.events`     |
